# Gold Fleet Backend (Laravel API)

A Laravel 12 REST API for the Gold Fleet Management System. This is the server-side backend that provides all business logic, database operations, and API endpoints.

---

## 🚀 Quick Start

### Prerequisites
- **PHP** 8.2+ 
- **Composer** (PHP dependency manager)
- **PostgreSQL** 12+ (running and accessible)
- **Node.js** & **npm** (for frontend assets)

### Get Started (30 seconds)
```bash
# Navigate to backend
cd backend

# Run automated setup (one time)
composer setup

# Start development environment (runs all services)
composer dev

# Access the API at http://localhost:8000
```

## 💾 Installation

### Option 1: Automated Setup (Recommended)

#### Windows
```bash
# Navigate to backend directory
cd backend

# Run setup script
setup.bat
```

#### Linux/Mac
```bash
# Navigate to backend directory
cd backend

# Make script executable and run
chmod +x setup.sh
./setup.sh
```

**What the setup script does:**
- ✅ Installs PHP dependencies (composer install)
- ✅ Creates .env configuration file
- ✅ Generates application key (APP_KEY)
- ✅ Runs database migrations (creates tables)
- ✅ Optionally seeds test data (php artisan db:seed)
- ✅ Creates storage symlink (for file uploads)
- ✅ Installs and builds frontend (npm install && npm run build)

### Option 2: Manual Installation

#### Step 1: Copy Environment File
```bash
cp .env.example .env
```

#### Step 2: Install PHP Dependencies
```bash
composer install
```

#### Step 3: Generate Application Key
```bash
php artisan key:generate
```

This generates a unique `APP_KEY` in your `.env` file for encryption.

#### Step 4: Configure Database (see Environment Configuration below)
Edit `.env` and set database credentials.

#### Step 5: Create Database
```bash
# Using PostgreSQL client
psql -U postgres -c "CREATE DATABASE Gold_Fleet;"

# Or let Laravel create it automatically during migration
php artisan migrate
```

#### Step 6: Run Database Migrations
```bash
php artisan migrate
```

This creates all required database tables.

#### Step 7: Create Storage Symlink (Required for File Uploads)
```bash
php artisan storage:link
```

#### Step 8: Seed Test Data (Optional, Development Only)
```bash
php artisan db:seed
```

This populates the database with sample vehicles, drivers, and users for testing.

## ▶️ Running the Application

### Option 1: All Services at Once (Recommended)

```bash
composer dev
```

This starts:
- Laravel API server (http://localhost:8000)
- Queue listener (background jobs)
- Application logs (real-time)
- Frontend Vite watcher (npm run dev)

All services run concurrently with color-coded output.

**Expected Output:**
```
[server]  Starting Laravel development server: http://127.0.0.1:8000
[queue]   Processing jobs...
[logs]    Listening for logs...
[vite]    VITE v7.2.4  ready in 234 ms
```

Press `Ctrl+C` to stop all services.

### Option 2: Individual Services

#### API Server Only
```bash
php artisan serve
```

Starts at http://localhost:8000

#### Frontend Development Server
```bash
npm run dev
```

Starts at http://localhost:5173

#### Queue Listener (Background Jobs)
```bash
php artisan queue:listen --tries=1
```

Listen for queued jobs.

#### Application Logs
```bash
php artisan pail --timeout=0
```

Stream real-time logs from the application.

## ⚙️ Environment Configuration

### Configure `.env` File

Copy `.env.example` to `.env` and update values:

```bash
cp .env.example .env
```

### Key Configuration Variables

#### Basic Application
```env
APP_NAME=Laravel              # Application name
APP_ENV=local                 # Environment: local, testing, production
APP_KEY=base64:xxxxx          # Generated by `php artisan key:generate`
APP_DEBUG=true                # Debug mode (false in production)
APP_URL=http://localhost      # Application base URL
```

#### Database Configuration (PostgreSQL)
```env
DB_CONNECTION=pgsql          # Database type: pgsql (PostgreSQL)
DB_HOST=127.0.0.1            # Database host (localhost)
DB_PORT=5432                 # PostgreSQL default port
DB_DATABASE=Gold_Fleet       # Database name to create/use
DB_USERNAME=postgres         # Database user
DB_PASSWORD=Christian12345@  # Database password (change in production!)
```

**Important**: Ensure PostgreSQL is running before migrations!

#### Session & Authentication
```env
SESSION_DRIVER=database       # Store sessions in database
SESSION_LIFETIME=120          # Session timeout (minutes)
SESSION_DOMAIN=localhost      # Domain for cookies
SESSION_SAME_SITE=none        # Allow cross-site cookies (for SPA)
SESSION_SECURE_COOKIE=false   # Use HTTPS only (false for local dev)
```

#### Sanctum (SPA Authentication)
```env
# Must include frontend URL so Sanctum treats requests as stateful
SANCTUM_STATEFUL_DOMAINS=localhost:5173

# This enables Sanctum to issue session cookies to the frontend
# Without this, the frontend cannot authenticate with the backend
```

#### File Storage
```env
FILESYSTEM_DISK=local         # Local storage (supports AWS S3, etc.)
APP_URL=http://localhost      # Base URL for storage links
```

#### Queue Processing
```env
QUEUE_CONNECTION=database     # Queue driver: database, redis, etc.
```

#### Caching
```env
CACHE_STORE=database          # Cache driver: database, redis, etc.
```

## 🗄 Database Setup

### Create Database

#### Using PostgreSQL Client
```bash
psql -U postgres
postgres=# CREATE DATABASE Gold_Fleet;
postgres=# \l              # List databases
postgres=# \q              # Exit
```

### Run Migrations

Migrations are in `database/migrations/`. They create database tables:

```bash
# Run all pending migrations
php artisan migrate

# Rollback last migration
php artisan migrate:rollback

# Reset database (rollback all, then re-migrate)
php artisan migrate:fresh

# Reset and seed (includes test data)
php artisan migrate:fresh --seed
```

### Seed Database (Test Data)

Seeders in `database/seeders/` populate the database with sample data.

```bash
# Run all seeders
php artisan db:seed

# Reset database and seed
php artisan migrate:fresh --seed
```

### View Database Tables

```bash
# Connect to PostgreSQL
psql -U postgres -d Gold_Fleet

# List all tables
\dt

# Exit
\q
```

## 🔐 Authentication (Sanctum)

Gold Fleet uses **Laravel Sanctum** for SPA (Single Page Application) authentication.

### How It Works

1. **Frontend (React)** sends login credentials to backend
2. **Backend** validates credentials and issues a session cookie
3. **Cookie** is automatically sent with all subsequent requests
4. **Backend** validates the cookie and processes the request

### Configuration for SPA

In `backend/.env`:
```env
# Allow Sanctum to create session-based cookies for the frontend
SANCTUM_STATEFUL_DOMAINS=localhost:5173  # Frontend URL

# Allow cookies across ports (local development)
SESSION_DOMAIN=localhost
SESSION_SAME_SITE=none                    # Cross-site cookies allowed
SESSION_SECURE_COOKIE=false               # HTTP is fine (not HTTPS)
```

### Protected API Calls

All API calls include the session cookie automatically (via `withCredentials: true` in Axios).

## 📤 File Uploads & Storage

### Storage Symlink Setup

File uploads are stored in `storage/app/public/` and accessed via `public/storage/`.

**Create symlink** (one-time setup):
```bash
php artisan storage:link
```

If symlink creation fails (especially on Windows):
```bash
# Verify symlink was created
dir public/
# Should show: storage/ [DIR]

# If not, create manually (Windows PowerShell as Admin)
New-Item -ItemType SymbolicLink -Path "public\storage" -Target "..\storage\app\public"
```

### Upload Files

#### From Backend
```php
// Store file in storage/app/public/vehicles/
$path = $request->file('image')->store('vehicles', 'public');
// Returns: 'vehicles/abc123.jpg'

// Get full URL for response
$url = Storage::url($path);
// Returns: '/storage/vehicles/abc123.jpg'
```

### Access Files

Files are served publicly at:
```
http://localhost:8000/storage/vehicles/abc123.jpg
```

## 🐛 Troubleshooting

### Issue: "SQLSTATE[08006] could not connect to server"
Database connection failed.

**Solution**:
1. Ensure PostgreSQL is running
2. Check `.env` database credentials
3. Verify database exists or create it
4. Restart PostgreSQL service

### Issue: "Port 8000 already in use"
```bash
# Find process
netstat -ano | findstr :8000

# Kill it
taskkill /PID <PID> /F

# OR use different port
php artisan serve --port=8001
```

### Issue: "Storage symlink does not exist" or File Upload Fails
```bash
php artisan storage:link
```

### Issue: "Class not found" or "Method not found"
Code changes not recognized.

```bash
# Clear all caches
php artisan cache:clear
php artisan config:clear
php artisan route:clear

# Regenerate autoloader
composer dump-autoload
```

### Issue: Tests fail or hang
```bash
# Clear test environment cache
php artisan cache:clear --env=testing

# Run without parallel (if hanging)
php artisan test --single-process
```
